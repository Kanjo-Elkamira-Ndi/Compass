# Database Schema

PostgreSQL 16 with the pgvector extension is the sole data store. All
schema objects are created and versioned through **Flyway** migration
files — never edit the schema by hand and never rely on
`hibernate.ddl-auto` beyond `validate` (it checks the schema matches the
entity mappings, it never modifies it).

## Conventions

- All PKs are `UUID`, generated by `gen_random_uuid()`.
- Every table has `created_at TIMESTAMPTZ DEFAULT NOW()`; most have
  `updated_at` managed by a trigger.
- Column names are `snake_case`; Java fields are `camelCase` (mapped by
  JPA/Hibernate, not manually).
- Entities never leave the service layer — see `architecture.md` §Layered
  request flow.

## Entity-relationship overview

| Relationship | Cardinality | Details |
|---|---|---|
| `User` → `Student` / `Lecturer` | 1:1 | Joined-table inheritance. `users` holds common fields; `students`/`lecturers` hold subtype fields, linked by shared PK. |
| `Student` ↔ `Course` | M:N | Resolved via `enrolments` join table (`enrolment_date`, `status`, `final_grade`). |
| `Lecturer` → `Course` | 1:N | Each course has one assigned lecturer, nullable until assigned. |
| `Student` → `GradeRecord` | 1:N | One record per student per course per semester. |
| `Student` → `PerformanceSummary` | 1:N | One summary per student per semester (computed and cached). |
| `Student` → `RiskAssessment` | 1:N | Multiple assessments over time; latest is current. |
| `User` → `ChatMessage` | 1:N | Grouped by `session_id` UUID, ordered by `created_at`. |
| `Lecturer` → `Exam` | 1:N | Each exam belongs to one lecturer and one course. |
| `Exam` → `ExamQuestion` | 1:N | Ordered list via `order_index`. |
| `User` → `ResearchAnalysis` | 1:N | Linked to the uploading user. |
| `Student` → `CareerRecommendation` | 1:N | Latest set is current; history retained. |
| `DocumentChunk` | — | Standalone RAG embeddings, no FK — documents are university-wide. |
| **`leads`** (public site) | — | Standalone, **no FK to `users`** — a visitor filling the contact form hasn't registered yet. |

### `exams` / `exam_questions`
`exams`: id, course_id (FK), lecturer_id (FK NOT NULL), title, topic,
difficulty (BEGINNER/INTERMEDIATE/ADVANCED), status (DRAFT/PUBLISHED).

`exam_questions`: id, exam_id (FK CASCADE DELETE), question_type
(MCQ/THEORY/PRACTICAL/CASE_STUDY), question_text, options (JSONB, MCQ
only: `[{label, text, correct}]`), correct_answer, order_index.

### `document_chunks` (pgvector RAG table)
| Column | Type | Constraints | Notes |
|---|---|---|---|
| id | UUID | PK DEFAULT gen_random_uuid() | |
| source_document | VARCHAR(255) | NOT NULL | Original PDF filename |
| page_number | SMALLINT | NULL | For citation |
| content | TEXT | NOT NULL | ~500-token chunk |
| embedding | vector(1536) | NOT NULL | `text-embedding-ada-002` output |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | |

> Index: `CREATE INDEX ON document_chunks USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);`
> `lists` ≈ `sqrt(row count)`.

### `revoked_tokens`
id (PK), token_hash (VARCHAR(64) UNIQUE NOT NULL — SHA-256 of the refresh
token), user_id (FK NOT NULL), revoked_at, expires_at (TTL; cleaned up by
scheduled job).

## Flyway migration sequence

| File | Description |
|---|---|
| V1…V8 | Core schema: users, students, lecturers, courses, enrolments, grade_records, performance_summaries, risk_assessments, chat_messages, exams, exam_questions |
| V9__create_security.sql | `revoked_tokens`, `password_reset_tokens` |
| V10__seed_admin.sql | Bootstrap admin user, reference data for programmes |
| **V11__create_leads.sql** | Public website `leads` table (additive, no dependency on core schema) |

When adding a new migration: next sequential `V{n}__description.sql`,
never edit a migration that has already run in any shared environment.
